下来我会在同一个对话里，分段生成一份抖音账号分析报告。你必须遵守以下全局规则与交付要求：

一、必须生成"图片 + Excel源文件"
- 需要产出 PNG 图片（在回复里直接展示预览 + 提供可下载文件）
- 需要产出 XLSX 源文件（可下载，且在 Excel 里能看到对应图表）

如果你当前无法生成文件：
1）请在回复第一行明确写：我当前无法直接生成PNG/XLSX文件（原因：…）
2）并改为提供：可复制CSV数据 + 一份完整可运行的Python脚本（读取我上传的xlsx/csv→生成png与xlsx）+ 3-5步运行说明
否则视为不合格输出。

二、数据输入
我会上传一个抖音账号逐条视频明细（xlsx/csv，1行=1条视频），至少包含：
- 发布时间（到秒或到分钟）
- 标题/文案
- 点赞、评论、收藏、转发（缺失列按0处理）

三、全局计算口径
- 互动总量 = 点赞 + 评论 + 收藏 + 转发
- 收藏率（%）= 收藏 / 互动总量 × 100

四、口径分区（非常重要）
- 第二节（月度趋势）只使用：按月统计"当月视频平均互动总量"，严禁在月度部分引入：阈值法 / P90 / 中位数 / MAD
- 第三节（爆款识别）才使用阈值法：
  当月阈值 = max(当月P90, 当月中位数 + 3×MAD)
  MAD = median(|x − median(x)|)
  爆款定义：单条视频互动总量 ≥ 当月阈值
  阈值法只用于：爆款清单、机制抽象、爆款选题库，不参与月度趋势计算

五、反硬凑
任何分期/分类/公式/数量目标都以数据是否支持为前提；不支持就说明原因并给替代输出，禁止为了完成结构硬凑。

六、防漂移（每一节都适用）
- 本节所有数字必须从原始数据重新计算，不得引用上一节的中间结果。
- 每节输出末尾必须附"字段映射与校验"：你识别到的点赞/评论/收藏/转发/发布时间/标题列名分别是什么；并任选1条视频手算一次互动总量与收藏率（写出计算过程与结果）。

七、重要：不要把"写作指令"原样输出到正文
- 提示词里出现的"必须/要求/结构/验收/交付物/字段/Sheet名"等说明，只用于你内部执行；
- 正文里禁止出现任何"【…】/（验收）/（交付物）/（结构说明）"这类字样；
- 正文只允许出现：章节标题 + 实际内容 + 表格 + 图片预览 + 下载链接 + 校验信息。

我会按顺序向你索取：S1账号概况、S2月度趋势、S3每日爆款、S4选题库、S5生成Word。你每次只输出我指定的那一节，不要提前输出后续内容。
